# Docker Containers

## What is a Container?

A Docker container is a **running instance of a Docker image**.

- **Image** = Blueprint (Template)
- **Container** = Running Instance

Containers provide an isolated environment where applications run consistently with all the required dependencies.

---

## Why Containers?

- Lightweight and fast to start.
- Run consistently across different environments.
- Isolate applications and their dependencies.
- Efficiently utilize system resources.
- Simplify application deployment and scaling.

---

## Container Lifecycle (With Example)

Let's practice using the **nginx** image.

---

### Step 1: Create a Container (Without Starting)

```bash
docker create --name mynginx nginx
```

Check:

```bash
docker ps -a
```

**Status → Created**

---

### Step 2: Start the Container

```bash
docker start mynginx
```

Check:

```bash
docker ps
```

**Status → Running**

---

### Step 3: Pause the Container

```bash
docker pause mynginx
```

**Status → Paused**

---

### Step 4: Unpause the Container

```bash
docker unpause mynginx
```

**Status → Running**

---

### Step 5: Stop the Container

```bash
docker stop mynginx
```

**Status → Exited**

---

### Step 6: Restart the Container

```bash
docker restart mynginx
```

Restarts the container by stopping it (if running) and starting it again.

---

### Step 7: Kill the Container

```bash
docker kill mynginx
```

Forcefully stops the running container immediately.

---

### Step 8: Remove the Container

```bash
docker rm mynginx
```

Permanently removes the container from the host.

# Working with Running Containers

## Run Nginx in Detached Mode

- **-d** → Run the container in the background.
- **-p** → Publish a container port to the host.

```bash
docker run -d -p 8080:80 --name webserver nginx
```

---

## Understanding Port Mapping (-p)

Port mapping allows applications running inside a container to be accessed from the host machine.

### Format

```text
HOST_PORT:CONTAINER_PORT
```

### Example

```text
8080:80
```

- **8080** → Host Port (Your Computer)
- **80** → Container Port (Inside the Container)

### Request Flow

```text
Browser
   │
localhost:8080
   │
Host Port (8080)
   │
Container Port (80)
   │
Nginx Container
```

**Left Side → Host Port**

**Right Side → Container Port**

## Passing Environment Variables

Environment variables allow configuration values to be passed into a container without modifying the application.

Use the **`-e`** option to define environment variables.

### Example 1: Pass a Single Variable

```bash
docker run -e APP_ENV=production nginx
```

This creates:

```text
APP_ENV=production
```

inside the container.

---

### Example 2: Pass Multiple Variables

```bash
docker run \
-e USERNAME=admin \
-e PASSWORD=1234 \
nginx
```

---

### Example 3: Verify Environment Variables

Run the container:

```bash
docker run -d --name envtest -e MY_NAME=Docker nginx
```

Enter the container:

```bash
docker exec -it envtest /bin/sh
```

Verify the variable:

```bash
echo $MY_NAME
```

Output:

```text
Docker
```

### Common Uses

- Database credentials
- API keys
- Application configuration
- Environment settings (Development, Testing, Production)

## View Logs

Display logs generated by the running container.

```bash
docker logs webserver
```

---

## View Real-Time Logs

Monitor logs continuously.

```bash
docker logs -f webserver
```

Press **Ctrl + C** to stop viewing logs.

---

## Execute Commands Inside a Container

Open an interactive shell inside a running container.

```bash
docker exec -it webserver /bin/sh
```

Example:

```bash
ls
cd /usr/share/nginx/html
ls
```

Type **exit** to leave the container.

---

## Run a Single Command

Execute a single command inside a running container without opening a shell.

```bash
docker exec webserver ls /
```

---

## Inspect a Container

Display detailed information about a container.

```bash
docker inspect webserver
```

The output includes:

- Container ID
- Container Name
- Image
- Container Status
- IP Address
- Port Mappings
- Environment Variables
- Mounted Volumes
- Network Configuration

---

## Key Takeaways

- A container is a running instance of a Docker image.
- Containers are lightweight, portable, and isolated.
- Containers move through different lifecycle states.
- Port mapping exposes container services to the host machine.
- Environment variables simplify application configuration.
- Logs and inspection help monitor and troubleshoot containers.
